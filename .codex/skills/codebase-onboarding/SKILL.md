# Codebase Onboarding

## Goals

- Fast architecture map
- Build/test/run entry points
- Risk zones and ownership hints

## Workflow

1. Scan repo signals
- Root files, package managers, CI workflows, infra configs.

2. Detect stack + runtime
- Primary languages, frameworks, build system, test framework.

3. Identify execution paths
- App entry points, API routes, job runners, migrations.

4. Capture contributor essentials
- Setup commands, lint/test commands, release flow.

5. Output onboarding brief
- "How to run", "Where to change", "What can break".

## Deliverable

A concise onboarding note usable by an engineer within first hour of touching the repo.
